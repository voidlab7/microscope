# MicroScope 工具选型（2026 年 4 月调研）

> 每个技能对应的最佳工具，经全网搜索对比后选定

---

## 总览

| 技能 | 推荐工具 | 备选 | 理由 |
|------|---------|------|------|
| **PDF 解析** | Marker | Docling / MinerU | 2026 最强 PDF→MD，表格+公式+图表全支持，10x faster than Nougat |
| **PDF 表格提取** | pdfplumber | Camelot / Tabula-py | 纯 Python，无需 Java，layout-first 提取最准 |
| **Excel 解析** | pandas + openpyxl | xlrd | pandas 读取 + openpyxl 引擎，覆盖 80% 需求 |
| **图片 OCR** | PaddleOCR | Surya / DocTR | 中文识别最强，表格识别内置，2026 OCR 王者 |
| **Word 解析** | python-docx | mammoth / bella-domify | 成熟稳定，表格+段落+样式全支持 |
| **网页抓取** | Firecrawl | Jina Reader / web_fetch | 专为 AI Agent 设计，返回清洁 Markdown |
| **网络搜索** | Tavily | Serper / Exa | AI 优化搜索结果，Agent 生态首选，MCP 支持 |
| **企业查询** | 天眼查 Open API | 企查查 API / web_search | 数据最全，API 规范，免费额度够用 |
| **报告生成 (MD)** | Jinja2 + 模板 | - | 模板引擎，灵活可控 |
| **报告生成 (Excel)** | openpyxl | xlsxwriter | 读写都支持，格式化丰富 |
| **报告生成 (PDF)** | markdown-pdf | ReportLab / WeasyPrint | MD→HTML→PDF 一站式 |
| **报告生成 (HTML)** | Jinja2 + 内联样式 | - | 复用 forum-html-writer 思路 |

## 详细对比

### 1. PDF 解析

| 工具 | 优势 | 劣势 | 适用场景 |
|------|------|------|---------|
| **Marker** ⭐ | 最准确的 PDF→Markdown，支持表格/公式/图表，10x faster than Nougat | 需要 GPU 加速（CPU 也能跑但慢） | 复杂报告（图表+表格+公式混排） |
| **pdfplumber** ⭐ | 纯 Python，表格提取精准，layout-first | 不处理 OCR（扫描件不行） | 原生 PDF 表格提取 |
| MinerU | 国产开源，中文优化 | 依赖较重 | 中文 PDF 优先 |
| Docling | IBM 出品，准确率高 | 安装复杂 | 企业级文档 |
| Nougat | Meta 出品，学术 PDF 强 | 慢，arXiv 训练集偏向 | 学术论文 |
| Camelot | 表格提取老牌 | 依赖 Ghostscript | 简单表格 |
| Tabula-py | Java 生态稳定 | 依赖 Java 运行时 | 不想用纯 Python 时 |

**选型决策**：Marker（通用解析）+ pdfplumber（精确表格提取）双引擎

### 2. 图片 OCR

| 工具 | 优势 | 劣势 | 适用场景 |
|------|------|------|---------|
| **PaddleOCR** ⭐ | 中文识别最强，内置表格结构识别，PP-Structure | 模型文件大（~100MB） | 中文文档/表格/截图 |
| Surya | 90+ 语言，速度快 | 中文不如 PaddleOCR | 多语言文档 |
| DocTR | 架构现代，准确率高 | 社区较小 | 英文文档 |
| Tesseract 5.5 | 老牌开源，100+ 语言 | 准确率不如深度学习方案 | 简单场景 |
| EasyOCR | 安装简单 | 速度慢，准确率中等 | 快速原型 |

**选型决策**：PaddleOCR（中文场景为主）

### 3. 网络搜索 API

| 工具 | 优势 | 劣势 | 价格 |
|------|------|------|------|
| **Tavily** ⭐ | 专为 AI Agent 优化，返回结构化结果，MCP 支持 | 国内访问需代理 | 免费 1000 次/月 |
| Serper | Google SERP 结果，便宜 | 返回原始 SERP，需自己解析 | $50/5000 次 |
| Exa | 语义搜索，找相似内容 | 通用搜索不如 Google | 免费 1000 次/月 |
| Firecrawl | 搜索+抓取一体 | 偏爬虫，搜索不是强项 | 免费 500 次/月 |
| web_search (内置) | CodeBuddy/OpenClaw 自带 | 无法精细控制 | 免费 |

**选型决策**：优先用平台内置 web_search，深度调研用 Tavily MCP

### 4. 企业信息查询

| 工具 | 优势 | 劣势 | 价格 |
|------|------|------|------|
| **天眼查 Open API** ⭐ | 数据最全（工商/融资/股东/诉讼/专利） | 需注册企业账号 | 免费 100 次/天 |
| 企查查 Open API | 数据全，API 设计好 | 商业 API 偏贵 | 按次收费 |
| 企知道 | 免费网页查询 | 无正式 API | 免费 |
| web_search 兜底 | 搜索公开报道和公告 | 数据碎片化 | 免费 |

**选型决策**：天眼查 API（结构化查询）+ web_search（新闻/动态补充）

### 5. 网页内容抓取

| 工具 | 优势 | 劣势 | 价格 |
|------|------|------|------|
| **Firecrawl** ⭐ | AI 优化，返回清洁 Markdown，JS 渲染 | 有速率限制 | 免费 500 次/月 |
| Jina Reader | URL→Markdown，简单好用 | 复杂页面可能丢内容 | 免费 |
| web_fetch (内置) | 平台自带 | 部分网站抓不到 | 免费 |
| Playwright | 浏览器级渲染 | 重，启动慢 | 免费 |

**选型决策**：优先 web_fetch（内置），失败时用 Firecrawl

## 安装命令

### Phase 1：核心依赖（Python）

```bash
pip install marker-pdf pdfplumber pandas openpyxl paddleocr python-docx jinja2
```

### Phase 2：增强依赖

```bash
pip install tavily-python firecrawl-py openpyxl xlsxwriter markdown-pdf
```

### MCP 工具（可选）

```json
{
  "tavily": {
    "command": "npx",
    "args": ["-y", "tavily-mcp"],
    "env": { "TAVILY_API_KEY": "your-key" }
  },
  "firecrawl": {
    "command": "npx",
    "args": ["-y", "firecrawl-mcp"],
    "env": { "FIRECRAWL_API_KEY": "your-key" }
  }
}
```
